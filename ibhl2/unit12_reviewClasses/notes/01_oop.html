D.1.1 Outline the general nature of an object.

Real-world objects are everywhere: your dog, your desk, your television, your bicycle, your wallet. Real-world objects share two characteristics: They all have state and behavior. Dogs have state (name, color, breed, hungry) and behavior (barking, fetching, wagging tail). Bicycles also have state (current gear, current pedal cadence, current speed) and behavior (changing gear, changing pedal cadence, applying brakes). The state of an object describes an object's attributes. The behavior of an object describes what it does.

D.1.2 Distinguish between an object(definition, template or class) and instantiation.

An object is defined by a class. A class is a description, model, or blueprint from which an object is created. Consider the blueprint created by an architect when designing a house. The blueprint defines the important characteristics of the house - its walls, windows, doors, electrical outlets, and so on. Once the blueprint is created, several houses can be built using it.

The houses built from the blueprint are different. They are in different locations, they have different addresses, contain different furniture, and different people live in them. Yet in many ways they are the "same" house. The layout of the rooms, number of windows and doors, and so on are the same in each. To create a different house, we would need a different blueprint. A class is a blueprint of an object. However, a class is not an object any more than a blueprint is a house.

Instantiation is process of using a class's constructor to create an object. To instantiate an object in Java you use the keyword new.

Student student = new Student("John", "Wayne");

D.1.5 Describe the process of decomposition into several related objects.

Class decomposition is the process of breaking a programming problem into multiple classes where each class has a specific task to perform in solving the problem.

Class Decomposition Example

Class decomposition can be illustrated through the problem of running a high school. Running a high school is a major undertaking. The first thing to be done in running a school is for the school board to hire a principal. The principal is the school's manager, he or she is responsible for everything that goes on in the school. The principal, however, cannot run the school alone, the job is just to big. So the principal hires people to help him. He hires assistant principals, counselors, teachers, librarians, coaches, secretaries, custodians, lunch workers, nurses, and the list goes on. Each person has a specific job to do in making the school run efficiently.

The structure chart for a high school might look like the following.

Class Decomposition

Hypothetically, if we were to write a computer program that solved the problem of running a high school each worker in the school would be a class. There would be a principal class, an assistant principal class, a counselor class, a teacher class, and so on. This is the process of class decomposition. Just as each worker in the school has a specific job to do, each class within the program has its own job to do in helping solve the problem.

D.1.7 Outline the need to reduce dependencies between objects in a given problem.

What is a Dependency?

Whenever a class A uses another class, then A depends on B. A cannot carry out it's work without B, and A cannot be reused without also reusing B. In such a situation the class A is called the "dependant" and the class B is called the "dependency". Two classes that use each other are said to be "coupled".

Why are Dependencies Bad?

Dependencies are bad because they decrease reuse. Decreased reuse is bad for many reasons. Often reuse has positive impact on development speed, code quality, code readability, code maintenance, etc.

How can Dependencies be Reduced?

One way object dependencies can be reduced is through the use of interfaces and abstract classes. Let's look at an example.

Suppose you have a method in a class named printList that takes an ArrayList as a parameter.

public void printList(ArrayList list)
{
    // print the list
}

Method printList is dependant upon the ArrayList class. The method is restricted to only working with ArrayList objects.

Now lets modify the method by replacing the ArrayList parameter with a reference to the List interface.

public void printList(List list)
{
    // print the list
}

By referencing the List interface the method is no longer "tightly coupled" to the ArrayList class. This makes the method more useful because it can now print any list that implements the List interface such as the LinkedList class.

D.1.9 Explain the need for different data types to represent data items.

A data type specifies the size and type of values that can be stored in an identifier or variable. The Java language provides 8 different primitive data types. Different data types allow you to select the type appropriate to the needs of the application. For example, if a program only needs variables for operations like counting or summing Java's integer types(int or long) will do, but if the program needs to store data like money that have decimal points it would need to use Java's real number types(float or double).

D.1.10 Describe how data items can be passed to and from actions as parameters.

Parameters

A parameter or argument is a value that is passed to a method so that the method can use that value in its processing. A parameter is located inside the parentheses that follow a method name.

The value 16 is a parameter or argument for the method sqrt. The method calculates the square root of this value and assigns it to the variable num.

A parameter can be a literal value.

Math.sqrt(81)
Math.round(76.9)
str.equals("exit")

A parameter can be a variable.

Math.sqrt(num)
Math.round(average)
str.equals(lastName)

A parameter can be a mathematical expression.

Math.sqrt(20 + 50 + 11)
Math.round((grade1 + grade2 + grade) / 3)

A parameter can be a call to another method.

Math.round(Math.sqrt(35))
Math.max(num3, Math.max(num1, num2))

A method can be have 0, 1, or more parameters. Multiple parameters are separated by commas.

Math.pow(10, 2)
Math.max(num1, num2)
str.substring(0, str.length()/2)

Writing a Method with Parameters

To add parameters to your own method definitions you simply declare one or more variables in the parentheses that follow the method name. These variable declarations need to include a data type and an identifier name. Look at the following example.

public double average(int gr1, int gr2, int gr3)
{
   return (gr1 + gr2 + gr3) / 3.0;
}

Method average declares three parameters all of type int named gr1, gr2, and gr3. These parameters are used to store the value of three grades that are passed to the method. The method uses these parameters to calculate the average of the three grades and returns the answer.

Parameter Scope

The scope of a parameter is limited to the method it is declared in. Look at the following example. 

public int minimum(int num1, int num2)
{
   if(num1 < num2)
     return num1;
   else
     return num2;
}

The parameters, num1 and num2, have method scope. This means they are local variables for the method minimum. They do not exist outside this method.


Formal vs Actual Parameters

There are two types of parameters: formal parameters and actual parameters or arguments. The parameters in the header of the method declaration are called formal parameters. The values passed into a method are called actual parameters or arguments.

D.2.3 Define the term polymorphism.

Polymorphism is a OOP feature that allows values of different data types to be handled using a uniform interface. This technique provides a consistent interface over a range of implementations.

Polymorphism is implemented in Java in two forms

  • Method Overloading (compile-time polymorphism) - defining more than one method in a class with the same name.
  • Method Overriding (run-time polymorphism) - the ability to redefine methods for derived classes(subclasses).

Method Overloading Example

// calculates and returns area of a rectangle
public double 
area
(int length, int width)
{  
    return length * width;
}

// calculates and returns area of a circle
public double 
area
(int radius)
{
    return Math.PI * radius * radius;
}

Two methods are defined both with the name area. The first method calculates the area of a rectangle and the second calculates the area of a circle.

Method Overriding Example

public class Hamburger
{
    public String 
getFood
()
    {
        return "hamburger";
    }
}

public class Meal extends Hamburger
{
    // override
    public String 
getFood
()
    {
        return super.getFood() + " french fries" + " drink"; 
    }
}

Method getFood is overridden (redefined) in the Meal subclass so that it also includes french fries and a drink.

D.2.4 Explain the advantages of encapsulation.

One aspect of encapsulation is the principle of information hiding. With information hiding objects hide their data (instance variables) from other classes by allowing access to the data only through the public methods defined within the class. Encapsulation therefore guarantees the integrity of the data contained in the object by controlling how the data is accessed and modified.

Benefits of Encapsulation

  • It simplifies the interaction between objects.
  • Allows a class to hide the internal details of how an object does something.
  • A class can change its internal implementation without hurting the overall functionality of the system.
  • Prevents code (data) from accidental corruption due to programming errors.

  • Easier Maintenance - code is well structured so it is easier to maintain.
D.2.6 Explain the advantages of polymorphism.

Polymorphism provides programmers the ability to treat objects of derived classes as if they were objects of the base class.

Benefits of Polymorphism

  • Common Interface - a programmer only has to learn one method name for a multitude of implementations.
  • Extensibility - a program that uses polymorphism is capable of handling any new requirements or enhancements that arise in the future due to its common interface.
  • Code Reuse - if an application does not use a common interface it will need to change and duplicate most of its code to support new implementations.
D.2.7 Describe the advantages of libraries of objects.

  • Efficient code - code in libraries is typically stable and optimized.
  • Code reuse
D.2.8 Describe the disadvantages of OOP.
  • Size - OOP programs are generally larger than normal programs.
  • Speed - OOP programs are generally slower than normal programs.
  • Effort - OOP programs require a lot more time and effort to create.
  • Learning - there is a larger learning curve when it comes to knowing how to develop OOP applications.

*In recent years the size and speed issues have been greatly improved, but the issue of effort still remains.

D.2.9 Discuss the use of programming teams.

A programming team is a team of people who develop or maintain computer software. The team programming approach has been utilized for many years but this approach has become even more important with the increased popularity of object-oriented programming. Many of the strategies used in a team programming approach involve multiple programmers working collaboratively on the same piece of source code as opposed to being individually responsible for individual tasks. Benefits of this approach include the ability for deficiencies in knowledge and ability in specific areas to be compensated for by the other programmer; in addition, the shared responsibility is thought to increase incentives for meeting project deadlines and quality targets. Therefore a team approach is more productive.
